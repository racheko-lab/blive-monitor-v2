小红书直播监控已于 2026-07-10 从代码中移除：开播短链每次都变、数据中心 IP 触发风控，维护成本高且误判/漏推难根除。当前专注 B站 / 抖音。
GitHub Token
用于前端直接写回 rooms.json / post_rooms.json（无需后端）。推荐用细粒度 Token，仅授权本仓库 racheko-lab/blive-monitor 的 Contents: Read & Write。
页面已内置一个默认 Token 以便开箱即用（无需配置即可增删监控）；如需更高安全性，可在此填入你自己的细粒度 Token 覆盖它。注意：内置 Token 暴露于公开源码，可随时在 GitHub 吊销轮换。
连接与检测
检测浏览器能否直连 api.github.com（中国大陆一般可直连）。
手动触发
立即让监控脚本跑一次（新增/移除房间或抖音号后会自动触发；此处可手动补触发）。
推送渠道
选择开播 / 新作品通知的推送方式。保存时配置会以加密方式写入仓库的 Actions Secret（BLIVE_CONFIG），不会出现在公开源码里（方糖每日限额 5 条，推荐 Bark / 企业微信 / PushPlus 不限量）。
定时摘要（A1）
开启后，直播视图顶部展示「今日/本周摘要」（N 人开播 · M 条新作），可一键复制或请求推送。自动投递由 CI 读取仓库 Secret（BLIVE_CONFIG.summary）完成。

推送文案模板（A4）
